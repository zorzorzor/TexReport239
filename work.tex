\section{Алгоритм решения задачи}


\subsection{Базовые структуры данных}

\par Понадобится 2 класса :
\par 1) Описывает поведение точки.
\par 2) Описывает поведение прямой.

\subsubsection {Первый класс (Point) :} 
\par Поля : "double x" - отвечает за координату точки по оси "x";
\par "double y" - отвечает за координату точки по оси "y".
\par 1) Сетеры(setX(),setY()).
\par 2) Гетеры(getX(),getY()).
\par 3) Метод "Distance Line(Line ln)" - возвращает расстояние от точки до данной прямой (ln).
\par 4) Конструкторы :
\par Point() - создает точку с координатами (0;0);
\par Point(double x,double y) - создает точку с координатами (x,y).
\par 5) toString - вывод точки в виде : *название точки* (x,y).

\par Т.к. класс не будет наследоваться, то все методы и поля приватные (private).

\subsubsection{Второй класс (Line) :}
\par Поля :
\par double k - отвечает за тангес угла прямой;
\par double b - отвечает за
\par длину отрезка, который отсекает прямая по оси "y", считая от начала координат.
\par 1) Сетеры(setX(),setY()).
\par 2) Гетеры(getX(),getY()).
\par 3) Метод "RelationPoint(Point p)" - возвращает "1", если точка лежит выше прямой, "0", если точка лежит на прямой, "-1", если точка лежит ниже прямой.
\par 4) Конструкторы : 
\par Line() - создает прямую "y=x";
\par Line(double k, double b) - создает прямую "y=kx+b";
\par Line(Point p1, Point p2) - создает прямую, которой принадлежат две данные точки (p1,p2).
\par 5) "toString" - вывод прямой в виде : *название прямой* y=k*x+b.

\par Т.к. класс не будет наследоваться, то все методы и поля приватные (private).



\subsection{Построение алгоритма}

\par Рассмотрим всевозможные прямые. Для каждой найдем разницу точек. Найдем минимальную разницу точек среди всех прямых - решим задачу

\par 1) Считываем количество точек (n) 
\par 2) Считываем координаты всех точек, создаем объект для каждой из них (с помощью массива объектов).

\par Всего возможно n*(n-1)/2 разных прямых (если учесть, что на любой прямой лежат только две точки из всех).

\par 3) Рассмотрим каждую прямую, создав цикл "for" :

\par !создавать каждый раз новую прямую не будем. создадим один раз, дальше будем только изменять ее коэффициенты!

\par Определим взаимное расположение точки, не принадлежащей рассматриваемой прямой, и этой прямой.
Заметим, что если сложить все значения этих точек "RelationsPoint" и взять модуль, то получим разницу количества точек, лежащих ниже прямой, и точек, лежащих выше прямой.
Соответственно, задача сводится к поиску минимума (для этого создадим переменную min) модуля суммы "RelationPoint" каждой точки к этой прямой (а для этого создадим переменную abs).

\par Для первой рассматриваемой прямой минимум и есть "abs", для остальных же, если min>abs, то min присваиваем abs, переменной "a" присвоем "getK()" прямой, переменной "b" присвоем "getY()" прямой (Нужно запомнить коэффициенты прямой, т.к на данный момент она искомая).

\par 4)Обнуление всех счетчиков, например abs, конец цикла.
\par 5)Вывод искомой прямой.
\par 6)Конец программы.
